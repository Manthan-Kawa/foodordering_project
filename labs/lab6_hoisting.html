Understanding JS Hoisting & Scope

1. Function Hoisting

Function declarations are hoisted - accessible before declaration
console.log(getOrderPrice(5)); // Works!

2. Variable Hoisting (var)

var is hoisted but initialized to undefined
console.log(coffeeType); // undefined
var coffeeType = "Espresso"; // Now assigned

3. let & const (Temporal Dead Zone)

let/const are hoisted but NOT initialized - ReferenceError before
// console.log(orderSize); // ReferenceError!
let orderSize = "Large";

4. Scope: var vs let

var has function scope, let has block scope
if(true) {
  var funcVar = 3.50; // Function scope
  let blockVar = "Small"; // Block scope
}
console.log(funcVar); // Works
console.log(blockVar); // ReferenceError

5. Function Expression

Function expressions are NOT hoisted (treated as var)
// calcTotal(); // TypeError: not a function
var calcTotal = function(x,y){ return x*y; };
